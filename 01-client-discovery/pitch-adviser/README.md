# Pitch-Adviser — сборка питч-документа для клиента

## Что это

Блок берёт готовые отчёты по клиенту (сайт + реклама), таблицу бенчмарков и
библиотеку наших кейсов — и собирает финальный HTML-документ для клиента:

1. Ваш сайт: какой трекинг стоит и что не работает (факты сканера).
2. Ваша реклама: какие объявления крутятся, какая аудитория (факты FB).
3. Прикидка затрат: сколько примерно потрачено (охват × частота × CPM).
4. Гэпы и рекомендации: что чинить в трекинге и что менять в рекламе.
5. «При тех же тратах было бы дешевле на X%» — аргумент для сделки.
6. 1–3 наших кейса, подобранных под клиента, как иллюстрация.

## Главный принцип

**Адвайзер работает только с готовыми, самодостаточными файлами. Минимум файлов.**
Здесь мы НЕ собираем и НЕ доделываем сырые данные. Если выхлоп сканера или
FB-скана недостаточен — это чинится на стороне движка (`engine/`), в отдельных
сессиях. Сюда записываем только TODO «движок должен отдавать файл такой-то»
(см. `todo/TODO.md`).

## Контракт входов

| Вход | Что это | Сколько файлов |
|---|---|---|
| Отчёт по сайту | итог сканера: страницы, пиксели, события, гэпы | 1 (максимум 3) |
| Отчёт по FB-рекламе | все активные объявления + демография + охваты | 1–2 |
| Google Ads (опция) | аналогично, когда дойдём | 1 |
| Бенчмарк-таблица | CPM, частота, worldwide-коэффициент — Google-спредшит | 1 |
| Библиотека кейсов | карточки кейсов в `cases/` | папка |

## Формула прикидки затрат

1. **Охват** = сколько уникальных людей увидели объявление.
   Meta раскрывает охват только по Европе и Великобритании —
   берём их одним числом: EU reach = Европа + Великобритания вместе.
2. **Частота** = сколько раз в среднем один человек увидел рекламу.
   Чем больше охват, тем ниже частота (небольшой охват → ~2–2.5,
   большой → ~1.5). Цифры — в бенчмарк-таблице.
3. **Показы** ≈ охват × частота.
4. **Затраты** ≈ показы ÷ 1000 × CPM (диапазон «от–до» из таблицы →
   оценка тоже диапазоном).
5. **Worldwide-прикидка**: если таргетинг — весь мир, EU+UK — лишь часть
   реальных трат. Исходим из данных FB об исключённых странах.
   Методика и коэффициент — после обсуждения (пока не зафиксированы).

В документе прикидка всегда помечается как оценка, не факт.

## Структура папки

```
pitch-adviser\
├── README.md        ← этот файл
├── pitch-decks\     ← PDF-деки с кейсами (сюда падают и новые)
├── cases\           ← нарезанные карточки кейсов + index.md (сводная таблица)
├── benchmarks\      ← ссылка на CPM-спредшит + правила (пока заглушки)
├── todo\TODO.md     ← припаркованные задачи
└── out\             ← готовые питч-документы по доменам
```

## Кейсы: упаковки и подбор

Деки — не разные наборы кейсов, а **разные упаковки одних и тех же**:
один кейс подаётся B2B-клиенту и B2C-клиенту с разных ракурсов (другие
акценты и цифры на плашках). Поэтому:

- **Тип клиента (B2B/B2C) — первичный фильтр подбора кейсов.** На входе
  получаем тип клиента → рассматриваем только кейсы с подходящей
  упаковкой → показываем страницы из соответствующего дека.
- Для B2C-подачи истина = свежий B2C-дек, для B2B-подачи = B2B-дек.

## Правило: новый PDF в pitch-decks\

Появился новый дек → раскладываем его на карточки кейсов тем же способом:
если это новая упаковка существующего кейса — дополняем его карточку,
если новый кейс — новый MD-файл в `cases/` (бренд, вертикаль, гео, канал,
суть, заявленные результаты, страницы по упаковкам, «когда показывать»)
плюс строка в `cases/index.md`. Карточки подтверждает Родион перед
использованием в питчах.
