## ✅ Детект дублирующих пикселей («жёлтая зона») — СДЕЛАНО

**Статус:** ✅ СДЕЛАНО 2026-06-23 · ветка `claude/hybrid-meta-dup-pixels` · реализовано ВМЕСТЕ с hybrid-Meta presence fix (один общий механизм — сбор pixel ID).
**Развилки решены:** выбран **generic** (ловит «два GA4» / «два Bing» тоже). Shopify Meta-пиксель остался известным ограничением (детект через web-pixels sandbox, не network `id=`).
**Бонус сверх плана:** тот же сбор ID закрыл Meta false-negative — пиксель виден по ID даже когда конверсионное событие не стрельнуло → статус «пиксель есть, конверсионное событие не зафиксировано» вместо ложного «NO TRACKING». (Прежняя формулировка «beacon в headless подавлён» — мисдиагноз, site-dependent; вычищено 2026-06-27, см. memory `project-headless-suppresses-meta-beacon`.)

### Зачем
Если на сайте стоит **больше одного пикселя одной платформы** (напр. два разных Meta
Pixel ID), это «жёлтая зона» — возможна дублирующая установка и **двойной счёт событий**
(один и тот же `Purchase` летит на два ID → ROAS в Ads Manager задвоен). Бывает и
легитимно (пиксель агентства + пиксель клиента, или Pixel + CAPI), поэтому это **не ошибка**,
а мягкая пометка-предупреждение.

### Почему сейчас не детектится
Сканер **не извлекает ID пикселя** — только метит «платформа Meta видна» и собирает события.
Два пикселя физически неотличимы от одного: оба бьют в `facebook.com/tr` и схлопываются
под общий ключ `"Meta"`. Чтобы ловить дубль, надо **начать собирать сам ID**
(число из `facebook.com/tr?id=ХХХХХ&ev=...`).

### Уровни детекта
- **На одной странице ≥2 ID** — сильный сигнал двойного счёта.
- **По сайту суммарно ≥2 ID** — слабее (может быть сегментация/миграция), но тоже помечаем.

### Известное ограничение

  network `id=`. Для него дубль так не поймать — отдельный кейс, в первый заход можно опустить.
