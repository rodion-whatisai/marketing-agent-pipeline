# AttentionNeeded — execution-формат триажа

> Короткий приоритизированный список «**что требует внимания и почему**», который отдаётся
> **агенту или человеку**. Сжимает портфель ad set'ов из живого дашборда (десятки строк ×
> 121 столбец ручного мониторинга) в несколько строк готового сигнала. Не принимает
> необратимых действий — только триаж; решение `scale / hold / pause` дальше, в
> [`engine/`](../engine/).

## Зачем это

В реальном FB-дашборде оператор каждый день глазами просматривает вкладку мониторинга:
по каждому ad set'у — факт-CPL, бенчмарк города, отношение факт/норма, тренд за 5 дней по 6
метрикам, «было ли включено вчера», и в конце ставит действие. Это **121 столбец**, по которым
человек сам ищет, что горит.

`AttentionNeeded` делает первый шаг за него и за агента: **Python переваривает таблицу в
ранжированный список** — кто над бенчмарком, насколько, куда идёт тренд, одной строкой «почему».
Потребитель (человек или агент-гейт) получает готовый сигнал, а не сырьё. Тот же принцип, что
в [`engine/`](../engine/): *LLM сырые числа читают плохо → отдаём переваренное*.

## Где сидит в конвейере

```
живой дашборд (вкладка мониторинга, ~18 ad set × 121 столбец)
        │   Python переваривает
        ▼
AttentionNeeded  ──►  агент  (вход для гейтов policy: gate-3 эффективность)
   (этот блок)   ──►  человек (короткий список «что горит сегодня»)
                          │
                          ▼
                 решение scale / hold / pause  (engine/policy.py)
```

## Поток (как из сырья приходим к выводу)

1. **читаю портфель** — по каждому ad set: город, спенд, лиды, факт-CPL, **бенчмарк-CPL города**,
   показы вчера, было ли включено вчера. *(`input/ad_set_performance.csv`)*
2. **считаю отношение** `ratio = факт-CPL / бенчмарк-CPL` — насколько ad set дороже/дешевле нормы
   своего города.
3. **читаю тренд** 5 дней CPL → направление (`rising / falling / flat`). Нули = дни без спенда
   (это не CPL) → отбрасываю. *(`input/cpl_trend_5d.csv`)*
4. **присваиваю severity** по бэндам и формулирую «почему» одной строкой.
5. **фильтрую и ранжирую**: оставляю `severity != ok`, сортирую severity → ratio убыв.

Пороги (**иллюстративные**, настраиваются под клиента):

| severity | условие (факт / бенч) | смысл |
|---|---|---|
| `high` | ≥ 1.30 | CPL на 30 %+ над нормой города |
| `medium` | 1.15 … 1.30 | +15…30 % |
| `watch` | 1.00 … 1.15 | 0…15 % над нормой — на контроле |
| `ok` | < 1.00 | дешевле нормы — внимания не требует |

## Результат демо (18 ad set → 7 требуют внимания)

`python attention_needed.py` → 2 `high` · 1 `medium` · 4 `watch` · 11 `ok`. Топ:

| severity | city | ad set | CPL | бенч | ratio | тренд | почему |
|---|---|---|--:|--:|--:|---|---|
| **high** | Dallas | `LG_DAL_Wide_EN` | $14.57 | $10.70 | 1.36 | flat | +36 % над нормой |
| **high** | Ottawa | `LG_OTT_Wide_EN` | $39.50 | $30.07 | 1.31 | **rising** | +31 %, тренд ухудшается |
| medium | Miami | `LG_MIA_Wide_EN` | $24.36 | $21.13 | 1.15 | falling | +15 %, выправляется |

Тренд — **второй сигнал поверх ratio**: Ottawa (high + растёт) хуже, чем Dallas (high, но ровно);
Miami (medium, но падает) уже выправляется. Это и есть «переваренный» вывод, который человек
иначе собирал бы глазами по 6 столбцам тренда.

## Про данные и честность

- Источник — **анонимизированный** срез реального FB lead-gen дашборда (7 городов).
  Имени клиента нет; имена ad set'ов (`LG_DAL_Wide_EN`) — generic ad-ops нотация.
- Колонка `action_human_ref` — что оператор реально сделал в исходнике — даётся **справочно,
  не как рекомендация**. В том срезе (конец месяца) оператор сворачивал почти всё, поэтому
  `off` стоит даже у дешёвых ad set'ов. `AttentionNeeded` ранжирует по факту vs бенчмарк —
  **воспроизводимо и без этого шума**.
- Пороги severity — иллюстративные. В проде калибруются по истории клиента (тот же
  rule-constructor, что в [`engine/rules.py`](../engine/rules.py)).

## Связь с engine/

`AttentionNeeded` — это вход в **gate-3 (эффективность)** policy-движка: `severity` ранжирует,
кого вообще смотреть, а `signal`/`rationale` — уже переваренный сигнал для решения. Сам
необратимый выбор (`pause_candidate` / `send_to_human`) остаётся за гейтами и человеком —
здесь только триаж.
